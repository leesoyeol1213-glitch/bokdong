## B. ✋ 업로드 키스토어 생성 (⚠️ 가장 중요 — 잃어버리면 앱 영구 업데이트 불가)

Play는 **서명된** 앱만 받습니다. 서명 키(키스토어)는 **service_role 키와 동급의 비밀** — 절대 레포/채팅에 올리지 마세요. `.gitignore`에 `*.jks`가 이미 등록돼 있습니다.

### B-1. 키스토어 생성 (내 PC에서 1회)
Claude Code 프롬프트에 `!` 붙여 실행하거나, PowerShell/터미널에서 직접:

```bash
keytool -genkeypair -v -keystore upload.jks -alias bokdong -keyalg RSA -keysize 2048 -validity 10000
```
- `keytool`은 JDK에 포함(Android Studio 있으면 이미 있음). 없으면 Temurin JDK 설치.
- 비밀번호 2개(키스토어/키) + 이름/조직 입력 → **이 비밀번호들 안전한 곳에 보관**(비밀번호 관리자 권장).
- 생성된 `upload.jks`를 **백업**(구글드라이브 비공개 등). 이거 잃으면 앱 업데이트 못 함.

> 참고: **Play 앱 서명**(Google Play App Signing)을 켜면, 위 키는 "업로드 키"가 되고 실제 배포 서명은 구글이 관리 → 업로드 키를 잃어도 재설정 가능(권장, 기본값).

### B-2. base64로 인코딩 (CI Secret용)
```bash
# Git Bash / Linux / Mac
base64 -w0 upload.jks > upload.jks.b64      # 이 파일 내용을 복사
# PowerShell
[Convert]::ToBase64String([IO.File]::ReadAllBytes("upload.jks")) | Set-Clipboard
```

### B-3. ✋ GitHub Secrets 등록
저장소 → **Settings → Secrets and variables → Actions → New repository secret** 로 4개 등록:

| Secret 이름 | 값 |
|---|---|
| `BOKDONG_KEYSTORE_BASE64` | 위 base64 문자열 전체 |
| `BOKDONG_KEYSTORE_PASSWORD` | 키스토어 비밀번호 |
| `BOKDONG_KEY_ALIAS` | `bokdong` (위에서 지정한 alias) |
| `BOKDONG_KEY_PASSWORD` | 키 비밀번호 |

## C. 🤖 릴리스 AAB 빌드 (제가 만든 워크플로 — 소열님은 실행만)

GitHub → **Actions → "Build Android Release (AAB)" → Run workflow** 클릭.
- 끝나면 **Artifacts → `bokdong-release-aab`** 다운로드 → 압축 풀면 `app-release.aab`.
- 이 파일을 Play Console에 업로드.
- (Secrets가 없으면 워크플로가 친절한 에러로 멈춥니다 — B단계 먼저.)

> ⚙️ 버전 올리기: 재출시할 때마다 `android/app/build.gradle`의 `versionCode`를 **+1**, `versionName`을 올려야 Play가 새 빌드로 인식합니다. (제가 커밋 때 같이 올려드릴 수 있음.)
