# 📊 PostHog로 "게임이 재밌는지" 읽는 법 (비개발자용)

> 계측은 이미 켜져 있어요(v9.64~). 이 문서는 대시보드에서 **뭘 보면 되는지**만 알려줍니다.
> 데이터는 사람이 실제로 플레이해야 쌓입니다. 며칠 모아서 보세요.

## 0. 먼저: 무시할 이벤트 2개
`bd_setup_verify`, `bd_browser_verify` — 제가 연결 확인용으로 보낸 거예요. 분석에서 무시하세요.

## 1. 우리가 기록하는 이벤트(퍼널 재료)
| 이벤트 | 의미 |
|---|---|
| `app_open` | 게임을 켰다 (`returning`=재방문, `pwa`=설치앱으로 실행) |
| `new_game` / `load_game` | 신규 시작 / 기존 진행 불러옴 |
| `onboarding_start` → `onboarding_done`/`skip` | 첫 환영 화면 진입 → 완료/건너뜀 |
| `ride_start` | **첫 라이딩 시작(활성화의 핵심!)** |
| `city_arrive` | 도시 도착(진행) |
| `bike_buy` / `gacha_roll` | 자전거 구매 / 가챠(현금·가챠권) |
| `level_up` / `prestige` | 성장 / 회차 환생 |
| `offline_reward` | 껐다가 다시 와서 오프라인 보상 받음(복귀 신호) |

## 2. 🎯 제일 먼저 볼 것 — 리텐션(계속 하는가)
왼쪽 메뉴 **Product analytics → Retention**
- **Cohortizing event**: `app_open`, **Returning event**: `app_open`
- 나오는 표: 첫날 켠 사람 중 **1일 뒤·7일 뒤 몇 %가 다시 켰나**.
- 방치형 게임 대략 기준(참고용): D1 20~30%면 양호, D7 10%대면 괜찮은 편. 낮으면 초반 재미/복귀 이유를 손봐야 함.

## 3. 🚀 활성화 퍼널 — 신규가 실제로 플레이까지 가는가
**Product analytics → Funnels**, 단계 순서대로:
1. `new_game`
2. `onboarding_done` (또는 `onboarding_start`)
3. `ride_start`
4. `city_arrive`
- 어느 단계에서 **뚝 떨어지는지**가 개선 포인트. 예: `ride_start`에서 많이 빠지면 "출발 버튼을 못 찾는다/이유를 모른다"는 뜻 → 온보딩·힌트 보강.

## 4. 💰 경제/재미 퍼널 (선택)
`ride_start` → `bike_buy` → `gacha_roll` → `prestige` 로 만들면
"얼마나 깊이 파고드는지(코어 몰입)"가 보여요.

## 5. 대시보드로 고정하기
각 인사이트에서 **Save** → **Add to dashboard**(예: "복동 핵심지표"). 매일 열어 추세만 봐도 충분.

## 6. 이렇게 쓰세요(루프)
데이터 관찰 → 약한 구간 1개 선택 → 그 부분만 개선 배포 → 다시 관찰. **한 번에 하나씩.**
"느낌"이 아니라 "숫자가 어디서 빠지나"로 결정하는 게 이 게임을 진짜로 키우는 방법이에요.
